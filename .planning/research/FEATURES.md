| Attendee list for organizer | Who signed up | Medium | Paid tiers universally |
| Ticket PDF / QR code generation | Check-in at door | Medium-High | TEC Event Tickets Plus, MEC Pro, Eventin Pro |

| Free tickets (RSVP) alongside paid | Mixed events | Low | Standard in ticketing plugins |

**Confidence:** HIGH for feature existence; MEDIUM for exact tier placement (may have shifted since training data).

## BuddyBoss-Specific Table Stakes

For a plugin sold as a BuddyBoss add-on, these integration points are what justify the product's existence. Without them, site admins would just use The Events Calendar.

| Feature | Why Expected | Complexity | Notes |
|---------|--------------|------------|-------|
| BuddyBoss group events tab | Groups are the primary organizing unit in BuddyBoss | High | No current competitor does this natively |
| Activity feed posting (event created, RSVP'd, ticket purchased) | BuddyBoss activity feeds are the community pulse | Medium-High | Requires BuddyBoss hooks |
| Member profile events section (hosted/attended) | Members' social identity on BuddyBoss is their profile | Medium | Requires BuddyBoss profile component hooks |
| Group member invite flow | Organizers want to reach their group, not send blast emails | Medium | Uses BuddyBoss group member API |
| BuddyBoss notification system integration | Members expect BB notifications, not just emails | Medium | BuddyBoss has its own notification layer |
| Respect BuddyBoss permission model | Admins, moderators, group organizers have different access levels | Medium | Must hook into BB roles/caps |

**Confidence:** HIGH — these are the BuddyBoss integration points explicitly called out in the PROJECT.md requirements. They are not offered by any competitor as native integration.

## Differentiators

Features that create competitive advantage. Not universally expected, but valued — especially for BuddyBoss community sites.

| Feature | Value Proposition | Complexity | Notes |
|---------|-------------------|------------|-------|
| Stripe Connect with platform commission (application fees) | Enables BuddyBoss to take a cut automatically without touching organizer funds | High | TEC, MEC, EventON do NOT do this — they have no platform layer |
| Tiered commission rates by BuddyBoss plan | Aligns plugin revenue with BuddyBoss's existing monetization model | Medium | Unique to this product |
| Admin-configurable event creation permissions (per plan tier) | Lets BuddyBoss site admins gate event creation as a premium feature | Medium | No competitor operates in this permission model |
| Recurring events with group-context awareness | Recurring events scoped to a specific group | High | Recurring events exist in TEC Pro / MEC; group-scoped recurrence does not |
| Virtual event support with external link (Zoom, Meet) | Hybrid community events are a real use case | Low-Medium | Most competitors support this but it is not always first-class |
| Event discovery across groups | See events from all groups you belong to, in one feed | Medium | Unique to BuddyBoss ecosystem; no competitor equivalent |
| "Invite from group roster" flow | Targeted invites without mass email | Medium | BuddyBoss-specific; no competitor equivalent |
| Co-organizer / event roles | Multiple people managing one event | Medium | TEC supports co-organizer concept; BuddyBoss group officers are natural co-organizers |
| Waitlist management | Fills cancellations automatically | Medium | TEC has a paid add-on; Eventin Pro includes it |
| Event check-in via QR code | Smooth door management | High | TEC Event Tickets Plus, Eventin Pro; differentiating if done better/cheaper |

**Confidence:** MEDIUM — competitive positioning based on training data. Verify that no competitor has released BuddyBoss integration since August 2025.

## Commission / Monetization Models Used by Competitors

This is relevant to the Stripe Connect + tiered commission model this plugin uses.

| Plugin / Platform | Commission Model |
|-------------------|-----------------|
| The Events Calendar | No platform commission — organizer keeps 100%. BuddyBoss Events would be differentiated here. |
| Eventin | No platform commission |
| MEC | No platform commission |
| EventOn | No platform commission |
| Eventbrite (non-WP, reference) | 3.7% + $1.79 per paid ticket (organizer fee) — similar application fee model |
| Ticket Tailor (non-WP, reference) | Flat per-ticket fee to platform |
| Amelia (WordPress booking plugin) | No platform commission — sold as one-time/annual license |
| Woo Tickets | No platform commission |

**Key insight:** No WordPress events plugin currently takes a platform commission from ticket sales. This model is borrowed from SaaS event platforms (Eventbrite, Ticket Tailor). It is genuinely novel in the WordPress ecosystem. This is both an opportunity and a risk — organizers accustomed to WordPress plugins that don't take a cut may resist.

**Mitigation:** The commission is justified by deep BuddyBoss integration, payment infrastructure, and plan-based discounts (higher plan = lower commission rate, aligning incentives).

**Confidence:** HIGH that no WP events plugin currently does platform commissions. MEDIUM on organizer resistance — based on general market reasoning, not primary research.

## Anti-Features (Deliberately NOT Build in v1)

Things to exclude from the first version to stay focused and reduce scope.

| Anti-Feature | Why Avoid | What to Do Instead |
|--------------|-----------|-------------------|
| Custom streaming / video infrastructure | Massive scope; Zoom/Meet/Teams already work | Accept external meeting URL field only |
| Offline / cash payments | Adds reconciliation complexity; out of scope per PROJECT.md | Stripe Connect only |
| Multi-currency support | Complex UX and tax implications | Single currency per organizer Stripe account |
| Event marketplace / public discovery (Eventbrite-style) | Not a BuddyBoss community need; completely different product | Events are scoped to the community, not a public marketplace |
| Native mobile app | Platform complexity; project explicitly out of scope | Responsive web — community members already use BuddyBoss web |
| AI-powered event recommendations | Premature; no event history data in v1 | Simple category filtering |
| Complex seating charts / venue maps | Very niche; high complexity for low initial ROI | Ticket quantity limits handle capacity |
| Affiliate / referral tracking for tickets | Feature creep in v1 | Defer to v2 based on organizer demand |
| Social login / standalone auth | BuddyBoss already handles auth | Rely entirely on BuddyBoss/WordPress user system |
| Sponsor management for events | Conference-level feature; not community events | Out of scope for v1 |
| White-label event pages (custom domain) | Technically complex; not a BuddyBoss community need | Standard WP template events pages |
| Bulk import / CSV upload of events | Edge case; adds import validation complexity | Manual creation + REST API for advanced users |

## MVP Recommendation

Prioritize for v1 (aligns with PROJECT.md active requirements):

1. **Event creation** — in-person, virtual, recurring (table stakes; entire product depends on it)
2. **BuddyBoss group events tab** — the core differentiator; without it this is just another events plugin
3. **Activity feed integration** — makes events feel native to the community, not bolted on
4. **Multiple ticket tiers + Stripe Connect** — the revenue mechanism; required for commission model
5. **Platform commission (application fees)** — BuddyBoss's monetization; required from day one
6. **Member profile events section** — social proof and community identity
7. **Group member invite flow** — key organizer workflow
8. **Admin control panel** — commission rates, permissions, payment settings
9. **Site-wide calendar view** — table stakes for any events plugin
10. **Admin-configurable creation permissions** — controls who can create events (critical for community moderation)

Defer to v2:
- QR code check-in (useful but not required for digital-first communities)
- Waitlist management (nice-to-have; manage with quantity limits in v1)
- Co-organizer roles (single organizer per event works for v1)
- Advanced recurring patterns (weekly/monthly covers 90% of cases)
- Event discovery feed across all groups (useful but complex; group tab covers v1 need)
