    Append six new functions to bp-events-functions.php AFTER the existing bp_events_format_notifications() function. Do not modify any existing function.

    1. bp_events_user_can_rsvp( $event_id, $user_id = 0 )
       - Returns WP_Error or true.
       - Checks: user is logged in, event exists and status='published', event is not cancelled.
       - Checks rsvp_group_id meta: if > 0, calls groups_is_user_member(); if false, returns WP_Error( 'bp_rest_events_rsvp_restricted', __('RSVP limited to members of this group.','buddyboss'), array('status'=>403) ).
       - If all checks pass, returns true.

    2. bp_events_rsvp_event( $event_id, $user_id = 0 )
       - Uses $wpdb and buddypress() global.
       - Defaults $user_id to bp_loggedin_user_id().
       - Returns false if !$user_id || !$event_id.
       - Calls bp_events_get_event(); returns false if event not found.
       - Counts registered rows: SELECT COUNT(*) FROM {$bp->events->table_name_attendees} WHERE event_id = %d AND status = 'registered'.
       - $at_capacity = !is_null($event->capacity) && $registered_count >= (int)$event->capacity.
       - $status = $at_capacity ? 'waitlisted' : 'registered'.
       - $wpdb->replace() into table_name_attendees with event_id, user_id, status, date_created=bp_core_current_time().
       - Returns $status on success, false on failure.

    3. bp_events_cancel_rsvp( $event_id, $user_id = 0 )
       - Defaults $user_id to bp_loggedin_user_id().
       - Checks current status of the row: SELECT status WHERE event_id=%d AND user_id=%d.
       - $was_registered = ('registered' === $current_status).
       - $wpdb->delete() from table_name_attendees WHERE event_id=%d AND user_id=%d.
       - If $was_registered && $result, calls bp_events_notify_waitlist( $event_id ).
       - Returns false !== $result.

    4. bp_events_get_attendees( $event_id, $status = 'registered' )
       - Returns array of stdObjects with user_id, status, date_created.
       - SELECT user_id, status, date_created FROM table_name_attendees WHERE event_id=%d AND status=%s ORDER BY date_created ASC.
       - Returns empty array if no rows.

    5. bp_events_get_waitlist( $event_id )
       - Wrapper: return bp_events_get_attendees( $event_id, 'waitlisted' ).

    6. bp_events_notify_waitlist( $event_id )
       - Gets waitlisted user_ids via SELECT user_id FROM table_name_attendees WHERE event_id=%d AND status='waitlisted'.
       - Returns early if empty.
       - Gets event via bp_events_get_event(); returns early if not found.
       - For each user_id: calls bp_notifications_add_notification() with component_name='events', component_action='waitlist_spot_open', allow_duplicate=false.
       - For each user_id: calls wp_mail() to $user->user_email with subject sprintf(__('A spot opened for %s','buddyboss'), $event_title) and body that includes event permalink and broadcast-model disclaimer (first to RSVP gets the spot).

    In register_routes(), after the existing gcal-url route registration, add three new register_rest_route() blocks:

    1. RSVP route — POST and DELETE on same pattern:
    ```php
    register_rest_route(
        $this->namespace,
        '/' . $this->rest_base . '/(?P<id>[\d]+)/rsvp',
        array(
            array(
                'methods'             => WP_REST_Server::CREATABLE,
                'callback'            => array( $this, 'rsvp_item' ),
                'permission_callback' => array( $this, 'rsvp_item_permissions_check' ),
            ),
            array(
                'methods'             => WP_REST_Server::DELETABLE,
                'callback'            => array( $this, 'cancel_rsvp_item' ),
                'permission_callback' => array( $this, 'rsvp_item_permissions_check' ),
            ),
        )
    );
    ```

    2. Attendees route — GET:
    ```php
    register_rest_route(
        $this->namespace,
        '/' . $this->rest_base . '/(?P<id>[\d]+)/attendees',
        array(
            'methods'             => WP_REST_Server::READABLE,
            'callback'            => array( $this, 'get_attendees' ),
            'permission_callback' => '__return_true',
        )
    );
    ```

    Add these four methods to the class body:

    rsvp_item_permissions_check( $request ):
    - Returns WP_Error if !is_user_logged_in() with 403.
    - Returns bp_events_user_can_rsvp( (int)$request->get_param('id') ) — propagates any WP_Error from the function.

    rsvp_item( $request ):
    - $event_id = (int)$request->get_param('id').
    - $result = bp_events_rsvp_event( $event_id ).
    - Returns WP_Error( 'bp_rest_events_rsvp_failed', ..., 500 ) if false.
    - Recomputes at_capacity: counts registered rows for the event, compares to capacity.
    - Returns new WP_REST_Response( array( 'status' => $result, 'at_capacity' => $at_capacity ), 200 ).

    cancel_rsvp_item( $request ):
    - $event_id = (int)$request->get_param('id').
    - $user_id = (int)$request->get_param('user_id') ?: bp_loggedin_user_id().
    - If $user_id !== bp_loggedin_user_id(): verify current user is event organizer ($event->user_id === bp_loggedin_user_id() or current_user_can('administrator')). Return 403 WP_Error if not.
    - $result = bp_events_cancel_rsvp( $event_id, $user_id ).
    - Returns new WP_REST_Response( array( 'cancelled' => $result ), 200 ).

    get_attendees( $request ):
    - $event_id = (int)$request->get_param('id').
    - $rows = bp_events_get_attendees( $event_id, 'registered' ).
    - Maps each row to array( 'user_id' => $row->user_id, 'display_name' => get_the_author_meta( 'display_name', $row->user_id ), 'avatar_url' => bp_core_fetch_avatar( array( 'item_id' => $row->user_id, 'type' => 'thumb', 'html' => false ) ), 'status' => $row->status ).
    - Returns new WP_REST_Response( $attendees, 200 ).
