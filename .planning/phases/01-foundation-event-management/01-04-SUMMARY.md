## Decisions Made

- **Moderation filter is defense-in-depth:** `bp_events_create_event()` already defaults status to 'pending' when moderation is enabled, but a direct `BP_Event::save()` call could bypass that. The `bp_events_before_event_save` hook ensures the rule is enforced at the persistence layer.
- **Child occurrences bypass moderation:** Occurrences are generated by the system when a parent is approved/published. Holding them pending would break recurring event generation.
- **`display()` override for inline JS:** The script block must appear once per table render. Overriding `display()` and calling `parent::display()` then `print_approve_script()` is the cleanest WP_List_Table extension point that guarantees co-location with the table HTML.
