<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
    Phase 3 BuddyBoss integration across four surfaces:
    - BB-01: Group Events tab with FullCalendar calendar, scoped to group events, privacy-enforced
    - BB-02: Activity feed items for event creation and RSVP, private groups hidden sitewide
    - BB-03: Group member invite panel on the event edit screen (when event has a group_id)
    - BB-04: Member profile attending and hosting tabs with event cards
  </what-built>
  <how-to-verify>
    Preconditions: MAMP is running, buddyboss-dev WordPress site is accessible, BuddyBoss Platform is active.

    --- BB-01: Group Events Tab ---

    1. Log in as an admin user.
    2. Visit a BuddyBoss group: http://buddyboss-dev.local/groups/{any-group-slug}/
    3. Confirm an "Events" tab appears in the group navigation bar.
    4. Click the Events tab. Confirm a FullCalendar calendar renders (not a white screen, not an error).
    5. Create a new published event associated with that group (or confirm an existing one is shown on the calendar).
    6. Confirm only that group's events appear on the calendar (not all site events).

    Privacy test (BB-01):
    7. Create or identify a PRIVATE group. Add a test event to that group.
    8. Log out or switch to a non-member account.
    9. Visit the private group URL. Confirm the Events tab is NOT visible (or is inaccessible) to non-members.

    --- BB-02: Activity Feed ---

    10. Log in as a member. Create a new published event (not associated with a group).
    11. Visit the sitewide activity feed: http://buddyboss-dev.local/activity/
    12. Confirm an activity item appears with text like "[User] created the event [Event Name]".

    13. As a different member, RSVP to that event.
    14. Confirm a new activity item appears: "[Member] RSVPed to the event [Event Name]".

    Privacy test (BB-02):
    15. Create a published event associated with a PRIVATE group.
    16. Log in as a member who is NOT in that private group.
    17. Visit the sitewide activity feed. Confirm the event_created activity item does NOT appear.
    18. Log in as a member who IS in the private group.
    19. Visit the group's activity feed (/groups/{slug}/activity/). Confirm the activity item DOES appear.

    --- BB-03: Group Member Invite ---

    20. Log in as an organizer. Create a new event and associate it with a group (set group_id on the event).
    21. After the event is created, navigate to its edit screen: http://buddyboss-dev.local/events/{event-slug}/edit/
    22. Confirm the "Invite Group Members" panel appears on the edit screen (the panel is NOT on the create screen — it appears here because the event's group_id is known).
    23. Confirm the panel loads a list of group members (names and avatars visible, not a loading spinner stuck).
    24. Type in the search box. Confirm the member list filters in real-time.
    25. Select 2-3 members. Confirm the selected count increments and the "Send Invites" button becomes enabled.
    26. Click "Send Invites". Confirm the status message says "Invites sent."
    27. In wp-admin or via WP-CLI, verify rows exist in wp_bp_event_invites with status='pending' for the selected users:
        `wp db query "SELECT * FROM wp_bp_event_invites WHERE event_id={event_id};" --path=/Applications/MAMP/htdocs/buddyboss-dev`

    --- BB-04: Member Profile Events ---

    28. Visit any member's profile: http://buddyboss-dev.local/members/{username}/
    29. Click the "Events" tab in the profile nav. Confirm sub-tabs "Attending" and "Hosting" are visible.
    30. Click "Attending". Confirm events the member has RSVPd to appear as event cards (same card style as the events directory).
    31. Click "Hosting". Confirm events the member has created appear as event cards.
    32. Visit a member who has NO events. Confirm the empty state message appears (not a blank or errored page).
  </how-to-verify>
